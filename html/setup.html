Setup
First-run wizard — prepare inventory, distribute SSH keys and define the topology. You can also come back here anytime to edit and download these files.

1. Inventory devices.yaml

Define every switch LLDPq should reach: management IPs (or hostnames), the default login username, and optional roles/groups. This is the source for collection, assets, key distribution and analysis.

2. SSH Keys

Authorize the collector's SSH key on every device in devices.yaml and enable passwordless sudo for the login user. This is the same single key pair used by Provision/ZTP.

Collector keychecking…
Key file—
If you have already distributed the key, just click Verify — no password needed. To push the key to devices that don't trust it yet, enter the switch admin password once: it is used transiently (ssh-copy-id) and is never stored. Already-trusting devices are skipped. Make sure Step 1 is saved first.

3. Topology topology.dot

Describe the expected cabling as a Graphviz .dot graph ("deviceA":"port" -- "deviceB":"port"). LLDP results are validated against this expected wiring.

4. Topology Config topology_config.yaml

Tune how the topology is rendered — tier ordering, which names are endpoint hosts, and special layout rules (e.g. staggered compute racks). Optional, but improves the Topology view.

5. Integrate Ansible optional

LLDPq can read your Ansible inventory to power the Ansible menu (VLAN Report, VRF Report, Fabric Exit/Config/Editor/Deploy). If you don't use Ansible, disable it here and those menu items disappear.

6. Run LLDPq

Run the full collection pipeline now (lldpq -: assets → LLDP check → monitor → fabric scan → alerts) and watch the live output. When it finishes, the Fabric, LLDP and Analysis pages are populated. It also runs automatically on the schedule below.

7. Update LLDPq

Pull the latest LLDPq and reinstall. Configs, keys and monitoring data are preserved across the update.

This is a Docker deployment. A container can't replace its own image, so run the update on the host that runs the container:
curl -fsSL https://aliaydemir.com/lldpq-amd64.tar.gz | gunzip | docker load && docker compose up -d

Loads the new image and recreates the container; named volumes (config & monitoring data) persist. Run it from the directory containing your docker-compose.yml.

This online update needs internet access to github.com — it runs git pull / git clone against the public LLDPq repository. On an air-gapped host (no internet / no GitHub access) this button won’t work; use an offline update instead.
Heads up: the update reinstalls and restarts nginx & fcgiwrap, so this page briefly disconnects and reconnects — the update keeps running in the background and the log resumes. Requires the LLDPq service account to have passwordless sudo.
