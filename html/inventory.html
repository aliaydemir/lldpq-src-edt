Inventory & Bootstrap
Upload the fabric design (P2P cabling + IP allocation), validate it, then generate the LLDPq install files. Every generated file is previewed and confirmed before it is written — nothing is applied silently.

1. Design Sources

Upload the P2P cabling workbook and the IPAM (IP allocation) workbook — .xlsx, .xlsm or a parsed connections.json. The newest uploads are kept as recoverable versions; pick one and click Use as Design to make it active. The active design is what the generators and Ask-AI use.

A wrong upload stays recoverable: the previous versions are retained automatically (up to the keep limit) as a single silent backup — there is no version-diff UI by design.

2. Validation Report

Deterministic design checks — duplicate ports, breakout mismatches, tbd/blank endpoints, duplicate cable/records. Runs against the active design; no AI required.

3. Generate topology.dot

Render the resolved P2P links as the expected topology. Preview first, then Set as expected topology writes topology.dot (the previous file is backed up to .bak).

4. Generate devices.yaml

Draft devices.yaml from the IPAM design — switch roles only (DGX/PDU/power/host filtered), SSH defaults preserved. Preview shows a diff against the current file before applying.

5. Generate topology_config.yaml

Draft topology_config.yaml (device_categories / layers / icons) from the roles present in the design. Preview, then confirm to write.

Bootstrap Advisor AI

Ask the advisor to propose or adjust artifacts — e.g. “propose topology.dot”, “remove leaf07 and add leaf09”, “generate devices.yaml”. Every suggestion still goes through the preview + confirm gate above; the advisor never writes install files.
